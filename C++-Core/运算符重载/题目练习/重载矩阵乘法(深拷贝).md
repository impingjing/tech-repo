# 重载矩阵的乘法运算

## 问题再现

> 
> 
> 
> 建立该整数矩阵类matrix构造函数；
> 
> 建立一个 *（乘号）的运算符重载，以便于对两个输入矩阵进行乘法运算；
> 
> 建立输出函数void display()，对整数矩阵按行进行列对齐输出；
> 
> 主函数里定义三个整数矩阵类对象m1、m2、m3.
> 
> ###输入格式：
> 
> 分别输入两个矩阵，分别为整数矩阵类对象m1和m2。
> 
> 每个矩阵输入如下：
> 
> 第一行两个整数 r  c，分别给出矩阵的行数和列数
> 
> 接下来输入r行，对应整数矩阵的每一行
> 
> 每行输入c个整数，对应当前行的c个列元素
> 
> ###输出格式：
> 
> 整数矩阵按行进行列对齐（宽度为10）后输出
> 
> 判断m1和m2是否可以执行矩阵相乘运算。
> 
> 若可以，执行m3=m1*m2运算之后，调用display函数，对m3进行输出。
> 
> 若不可以，输出"Invalid Matrix multiplication!"
> 
> 提示：输入或输出的整数矩阵，保证满足row>=1和column>=1。
> 

## 程序实现

```cpp
#include <iostream>
#include <iomanip>
#include <stdexcept>
using namespace std;

class Matrix{
private:
    int row;
    int column;
    int **mat;

    void allocate(int r,int c){
        row=r;
        column=c;
        if(r>0 && c>0){
            mat=new int*[r];
            for(int i=0;i<r;i++){
                mat[i]=new int[c];
                for(int j=0;j<c;j++){
                    mat[i][j]=0;
                }
            }
        }
        else{
            mat=nullptr;
        }
    }
    void distroy(){
        if(mat!=nullptr){
            for(int i=0;i<row;i++){
                delete[] mat[i];
            }
            delete[] mat;
        }
    }
public:
    //构造函数
    Matrix(int r=0,int c=0){
        allocate(r,c);
    }
    //析构函数
    ~Matrix(){
        distroy();
    }
    //拷贝构造函数
    Matrix(const Matrix& other){
        allocate(other.row,other.column);
        if(row>0 && column>0){
            for(int i=0;i<row;i++){
                for(int j=0;j<column;j++){
                    mat[i][j]=other.mat[i][j];
                }
            }
        }
    }
    //重载=
    Matrix& operator=(const Matrix& other){
        allocate(other.row,other.column);
        if(row>0 && column>0){
            for(int i=0;i<row;i++){
                for(int j=0;j<column;j++){
                    mat[i][j]=other.mat[i][j];
                }
            }
        }
        return *this;
    }
    //打印信息
    void display(){
        for(int i=0;i<row;i++){
            for(int j=0;j<column;j++){
                cout << setw(10) << mat[i][j];
            }
            cout << endl;
        }
    }
    //重载*
    Matrix operator*(const Matrix& other){
        if(column!=other.row){
            cout << "Invalid Matrix multiplication!"; exit(0);
        }
        else{
            Matrix temp(row,other.column);
            if(temp.row>0 && temp.column>0){
                for(int i=0;i<row;i++){
                    for(int j=0;j<other.column;j++){
                        int t=0;
                        for(int k=0;k<column;k++){
                            t += mat[i][k]*other.mat[k][j];
                        }
                        temp.mat[i][j]=t;
                    }
                }
            }
            return temp;
        }
    }
    //获得数组信息
    int** getMat(){
        return mat;
    }
    friend istream& operator>>(istream& in,Matrix& m);
};
//重载输入>>
istream& operator>>(istream& in,Matrix& m){
    if(m.row>0 && m.column){
        for(int i=0;i<m.row;i++){
            for(int j=0;j<m.column;j++){
                in >> m.mat[i][j];
            }
        }
    }
    return in;
}
int main(){
    int r1,c1,r2,c2;

    cin >> r1 >> c1;
    Matrix m1(r1,c1);
    cin >> m1;
    cin >> r2 >> c2;
    Matrix m2(r2,c2);
    cin >> m2;

    Matrix m3;
    m3 = m1*m2;
    m3.display();

    return 0;
}
```

## 疑难注意

### 1. 释放内存注意检查空指针

```cpp
~Matrix()
{
    if (mat != nullptr) { // 👈 增加空指针检查
        for(int i=0; i<row; i++){
            delete[] mat[i];
        }
        delete[] mat;
    }
}
```

### 2. 内存泄漏与浅拷贝的隐患

如果不写拷贝构造函数（深拷贝）和赋值运算符的重载（深拷贝），那么我们就会遇到：

**现象**：在 `m3 = m1 * m2;` 这一行，程序会发生**内存泄漏**，甚至崩溃。
**原因**：
`m1 * m2` 返回一个**临时的**矩阵对象（假设叫 temp）。
执行 **m3 = temp** 时，因为你没有重载赋值运算符 (operator=)，编译器会使用默认的 浅拷贝。
默认浅拷贝会执行 `m3.mat = temp.mat`。
**后果**：
-`m3` 原本在构造时分配的内存（虽然这里是 `0x0`，但如果是非空矩阵就严重了）没有释放，造成内存泄漏。
-`m3` 和 `temp` 指向同一块内存。语句结束后，`temp` 被销毁，调用析构函数 `delete[]` 了这块内存。`m3.mat` 变成了悬空指针（指向已被释放的内存）。
当 `main` 结束，`m3` 析构时，再次 `delete[]` 同一块内存，导致 **Double Free** 崩溃。

<aside>
💡

**三法则**是 C++ 里关于资源管理的一条经典经验法则。简单来说，它规定：如果你的类需要**手动管理资源**（比如你代码里的 new 和 delete），那么当你显式定义了**析构函数、拷贝构造函数、拷贝赋值运算符**这三者中的任意一个时，通常就需要把另外两个也一并定义出来。
