# 运算符重载

## （零）引入：

对于内置数据类型，编译器指导如何进行运算，但两个由类创建的对象能否进行加减运算呢？事实上，在运算符重载的作用下，我们可以用“+”等运算符直接对对象进行操作。

## （一）operator+

```cpp
class Person{
   // 如果用全局函数重载，我们需要声明全局函数为友元，以便全局函数可以访问类的私有成员
   friend Person operator+(Person &p1,Person &p2)；
   
private:
    int a;
    int b;
public:
    // 默认构造函数
    Person(int=1,int=2){}
    // **通过成员函数实现“+”的重载**
    Person operator+(Person &p){
        Person temp;
        temp.a=this->a+p.a;
        temp.b=this->b+p.b;
        return temp;
     }
 };
 /*
 **// 通过全局函数实现“+”重载**
 Person operator+(Person &p1,Person &p2){
     Person temp;
     temp.a=p1.a+p2.a;
     temp.b=p1.b+p2.b;
     return temp;
 }
 // 函数重载
Person operator+(Person &p,int i){
     Person temp;
     temp.a=i+p.a;
     temp.b=i+p.b;
     return temp;
 }
 // 如何调用？
 Person n_p=operator+(p1,p2);
 简化为：
 Person n_p=p1+p2;
 Person nn_p=p1+10;
 */
 int main(){
     Person p1(2,3);
     Person p2(4,5);
     
     // Person n_p=p1.operator+(p2) 简化为：
     Person n_p=p1+p2;
     
     return 0;
 }
```

## （三）operator++

```cpp
// 通过重载递增运算符，实现自己的整数类型
class MyInteger{
public:
    MyInteger(){
        value=0;
    }
    // 重置前置++
    MyInteger& operator++(){
        //先++
        value++;
        //再将自身做一个返回
        return *this;
        /*为什么不返回值？
        如果返回值，那么我们是无法对同一个对象进行多次递增操作。*/
    }
    // 重置后置++
    MyInteger operator++(int){ //此处的“int”作为占位参数，用于区分前置和后置
        // 先 记录当时结果
        MyInteger temp=*this;
        // 后 递增
        value++;
        // 最后将记录结果返回
        return temp;
        /* 为什么返回值？
        如果返回引用，由于temp是局部变量，编译器会报错。*/
private:
    int value;
};

int main(){
    MyInteger my_int;
    // 能够正常打印
    cout << my_int << endl;
    // 能够进行前置++操作
    cout << ++my_int << endl;
    // 能够进行后置++操作
    cout << my_int++ << endl;
    cout << my_int << endl;
    return 0;
}
```

## （四）operator=

### -引入：

C++编译器至少给一个类添加4个函数

1. 默认构造函数（无参，函数体为空）
2. 默认析构函数（无参，函数体为空）
3. 默认拷贝构造函数，对属性进行值拷贝
4. 赋值运算符operator=，对属性进行值拷贝

### -正文：

```cpp
class Person{
public:
    Person(int age){
        m_Age=new int(age); //动态开辟
    }
    ~Person(){
        if(m_Age!=NULL){
            delete m_Age; //手动释放
            m_Age=NULL;
        }
    }
    int *m_Age;
};

int main(){
    Person p1(18);
    cout << *(p1.m_Age) << endl;
    
    Person p2(20);
    p2=p1; //赋值操作
    cout << *(p2.m_Age) << endl;
    /* 没写析构函数前，上述输出可以正常运行；加了析构函数后，程序崩溃。为什么呢？
       该程序中的浅拷贝导致同一块区域被释放了两次。*/
    return 0;
} 
```

```cpp
// 在Person中写
// 重载赋值运算符
void operator=(Person &p){
    // 先判断是否有属性在堆区，如果有先释放干净，然后再深拷贝
    if(m_Age!=NULL){
        delete m_Age;
        m_Age=NULL;
    }
    // 做深拷贝操作，重新开辟一个空间
    m_Age=new int(*p.m_Age);
}
// 上述程序存在不足：无法实现连续赋值(类似于a=b=c)
Person& operator=(Person &p){
    if(m_Age!=NULL){
        delete m_Age;
        m_Age=NULL;
    }
    m_Age=new int(*p.m_Age);
    // 返回对象本身
    return *this;
}
// 这时就可以实现p1=p2=p3这种连等操作
```

## （五）operator==

```cpp
class Person{
private:
    int age;
    string name;
public:
    Person(){}
    Person(int a,string n):age(a),name(n){}
    
    // 重载关系运算符
    bool operator==(const Person &p)const{
        if(this->age==p.age && this->name==p.name)
            return true; 
        else
            return false;
     }
};

int main(){
    Person p1(18,"Amy");
    Person p2(18,"Amy");
    
    if(p1==p2) //重载了==,自定义类型可以直接进行比较
        cout << "SAME" << endl;
    return 0;
}
```

## （六）operator()

### -引入：

1. 函数调用运算符()也可以重载
2. 由于重载后使用的方式非常像函数的调用，因此称为仿函数
3. 仿函数没有固定写法，非常灵活

### -正文：

```cpp
class MyPrint{
public:
    void operator()(string test){
        cout << test << endl;
    }
};

void test(){
    MyPrint my_print;
    my_print("hello,you"); //“仿函数”
}
void _test(){
    MyPrint()("hello,me"); //匿名对象
}
int main(){
    test();
    return 0;
}
```

## （七）operator>>

### -引入：

#### 为什么不能作为成员函数？

如果将 `operator>>` 定义为类的成员函数，调用时的左操作数必须是该类的对象 。

- **成员函数形式：** `obj.operator>>(cin)` $\rightarrow$ 调用写法：`obj >> cin;`（这不符合习惯）。
- **全局函数形式：** `operator>>(cin, obj)` $\rightarrow$ 调用写法：`cin >> obj;`（符合标准用法）。

因此，为了让 `std::istream` 对象（如 `cin`）出现在 `>>` 的左侧，我们必须定义一个非成员函数 。

### -正文：

```cpp
class Complex {
private:
    double real, imag;

public:
    // 构造函数
    Complex(double r = 0, double i = 0) : real(r), imag(i) {}

    // 声明友元函数
    friend std::istream& operator>>(std::istream& in, Complex& c);
};

// 在类外实现全局重载函数
std::istream& operator>>(std::istream& in, Complex& c) {
    // 逻辑：从输入流中提取数据并存入对象的成员
    in >> c.real >> c.imag; 
    
    // 关键：必须返回输入流对象的引用，以支持链式调用（如 cin >> a >> b）
    return in; 
}
```

---

#### 核心要素拆解

**A. 参数设计**

- **第一个参数 (`std::istream& in`)**：必须是输入流对象的**引用**。因为输入流的状态在读取过程中会改变（如记录当前读取位置、处理错误标志等），且 `istream` 对象禁止拷贝 。
- **第二个参数 (`Complex& c`)**：必须是目标类对象的**非 const 引用**。因为输入操作的本质就是修改该对象的内容 。

**B. 返回值设计**

- **返回类型 (`std::istream&`)**：必须返回传入的那个输入流对象的引用 。
- **目的**：支持**链式输入**。例如 `cin >> c1 >> c2;` 等价于 `(cin.operator>>(c1)).operator>>(c2);` 。

---

#### 易错点与注意事项

- **不要在重载函数里写 `cout`**：输入重载函数应当只负责“读数据”。不要在里面写类似 `cout << "Please input:"` 的提示语，否则在处理文件流或自动化测试时会产生多余输出。
- **错误处理**：高级实现中，通常会检查输入流的状态（如 `if (in.fail())`），如果用户输入了非法字符（比如要求输入数字却输入了字母），需要进行相应处理。
- **深拷贝与输入**：如果类中包含指针成员，输入重载时通常需要先释放旧内存，再根据读入的长度申请新内存（类似于赋值重载的逻辑），以确保**深拷贝策略**的完整性 。
