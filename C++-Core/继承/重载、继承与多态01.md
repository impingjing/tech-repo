# 重载、继承与多态01

---

## 基础概念

### 重载

- 所有运算符，即可以重载为类的成员函数，也可以重载为全局函数，并往往声明为友元函数。❌️

<aside>
💡

有些运算符（**=、()、[]、→**）必须重载为成员函数，有些（**<<、>>**）只能重载为全局函数；另外，全局函数往往需要友元才能访问私有成员，但不是“往往声明为友元”这种绝对化表述（比如，全局函数只需要访问类的公有成员，此时可以不必进行友元声明）。

</aside>

- **不允许重载**的运算符有：`.`（成员访问运算符）`::`（作用域解析运算符）`?:`（三目运算符）`sizeof`（计算字节大小的运算符）`.*`（成员指针访问运算符）。✅️
- 要使重载的 `[]` 运算符能够出现在赋值表达式的左右两边，其返回值必须是**可被修改的左值引用**，即 `A&`。✅️

<aside>
💡

值拷贝是临时对象，无法作为左值被赋值。

</aside>

- 对单目运算符重载为友元函数时，可以说明一个形参。而重载为成员函数时，不能显式说明形参。✅️

<aside>
💡

友元函数（全局函数）重载单目运算符时，需要一个参数（对象本身）。成员函数重载单目运算符时，不需要显示参数，因为隐含的`this`指向对象。

</aside>

### 有关构造与析构的顺序

- 在多继承中，派生类的构造函数需要依次调用其基类的构造函数，调用顺序取决于定义派生类时所指定的各基类的顺序。✅️
- 封闭类（包含成员对象的类）构造时，先调用成员对象的构造函数，再进入封闭类的构造函数体。✅️
- 成员对象的初始化顺序只有它们在类中声明的顺序决定，与成员初始化列表中的顺序无关。✅️
- 在创建派生类对象时，构造函数的执行顺序是：基类构造函数→对象成员构造函数→派生类本身的构造函数。✅️

### 多态

- 类的构造函数可以定义为虚函数。❌️

<aside>
💡

构造函数不能是虚函数！因为虚函数机制依赖虚表指针（vptr），而vptr是在构造函数中初始化的。

</aside>

- 静态成员函数不能是虚函数，所以不能实现动态多态。✅️

<aside>
💡

静态成员函数属于类而不属于对象，没有this指针，无法参与动态绑定。

**动态绑定（多态）** 的核心逻辑是：基类指针指向派生类对象，调用虚函数时，程序通过对象内部的 **虚函数表指针（vptr）** 找到对应的 **虚函数表（vtable）**，从而决定调用哪个版本的函数。

 既然静态函数调用时根本不看对象（也就看不到对象里的 vptr），那它怎么可能根据对象的实际类型去跳转函数呢？

</aside>

- 通过基类指针调用基类和派生类中的同名虚函数时，若该指针指向的是一个基类的对象，那么调用的就是基类的版本。✅️

<aside>
💡

动态多态取决于指针实际指向的对象类型。

</aside>

- 抽象类不可以实例化对象，派生类如果不实现**所有**纯虚函数，那么它依然是一个抽象类。✅️
- 纯虚函数可以有函数体。✅️

### 继承

- 派生类对象可以被基类对象初始化。❌️
- 派生类对象可以隐式转换为基类对象。✅️
- 派生类对象可以初始化基类的引用。✅️
- 派生类指针可以隐式转换为基类指针。✅️

```cpp
class Base{
public:
    int b=1;
    void show(){……}
};
class Derived:public Base{
public:
    int d=2;
    void show(){……}
};

// 派生类对象->基类对象
Derived son;
Base father=son; //合法，但发生切片
//只把son中属于Base的部分拷贝给了father,son.d丢失;
//后续father调用show,依旧调用的是Base::show()

// 派生类对象->初始化基类引用
Derived d;
Base& rb=d; //合法，不切片
//rb依然引用原来的d对象；
//如果把Base中的show定义为虚函数,那么就可以实现多态

// 派生类指针->基类指针
Derived* pd=new Devived();
Base* pb=pd; //合法，向上转型
//pb指向Derived对象
//调用show的情况与上一个同理
```

## 程序问题

### 题目再现

> 定义抽象基类Shape，由它派生出五个派生类：Circle（圆形）、Square（正方形）、Rectangle（ 长方形）、Trapezoid （梯形）和Triangle （三角形），用虚函数分别计算各种图形的面积，输出它们的面积和。要求用基类指针数组，每一个数组元素指向一个派生类的对象。PI=3.14159f，单精度浮点数计算。
> 

### 代码实现

```cpp
#include <iostream>
#include <iomanip>
using namespace std;

const float PI=3.14159f;

class Shape{
public:
    virtual float area()=0;
    virtual ~Shape()=default;
};
class Circle:public Shape{
private:
    float r;
public:
    Circle(float _r=0):r(_r){}
    float area(){
        return PI*r*r;
    }
};
class Square:public Shape{
private:
    float a;
public:
    Square(float _a=0):a(_a){}
    float area(){
        return a*a;
    }
};
class Rectangle:public Shape{
private:
    float a;
    float b;
public:
    Rectangle(float x=0,float y=0):a(x),b(y){}
    float area(){
        return a*b;
    }
};
class Trapezoid:public Shape{
private:
    float a;
    float b;
    float h;
public:
    Trapezoid(float x,float y,float z):a(x),b(y),h(z){}
    float area(){
        return (a+b)*h/2;
    }
};
class Triangle:public Shape{
private:
    float a;
    float h;
public:
    Triangle(float x,float y):a(x),h(y){}
    float area(){
        return a*h/2;
    }
};

int main(){
    // 创建基类指针数组
    Shape* shape[5];
    float c_r,s_a,r_a,r_b,t_o,t_u,t_h,tr_a,tr_h;
    cin>>c_r>>s_a>>r_a>>r_b>>t_o>>t_u>>t_h>>tr_a>>tr_h;
    // 实例化对象
    shape[0]=new Circle(c_r);
    shape[1]=new Square(s_a);
    shape[2]=new Rectangle(r_a,r_b);
    shape[3]=new Trapezoid(t_o,t_u,t_h);
    shape[4]=new Triangle(tr_a,tr_h);
    // 输出所有图形的面积和
    float sum_area=0;
    for(int i=0;i<5;i++){
        sum_area += shape[i]->area();
    }
    cout << fixed << setprecision(3) << sum_area << endl;
    // 销毁，防止内存泄漏
    for(int i=0;i<5;i++){
        delete shape[i];
    }
    return 0;
}
```

### 疑难注意

- 对临时对象取地址。❌️
    
    ```cpp
    shape[0]=&Circle(c_r);
    	// Circle(c_r)创建了一个临时对象，表达式结束后就会被销毁
    	// 你用&取它的地址，但这个地址在下一行代码执行时就无效了
    //正确做法✅️：
    shape[0]=new Circle(c_r);
    ```
    
- 对局部数组进行delete操作。❌️
    
    ```cpp
    // 你可能会想，既然我们用来new，那么就必须用delete来防止内存泄漏；
    // 而shape正好是一个数组，所以写了这样一行代码：
    delete[] shape;
    // 错误！为什么？因为 Shape* shape[5]; 这种开辟方式说明shape是栈上的固定数组
    // new开辟的是shape数组中的每个元素，而不是shape数组本身
    // 故我们要一个元素一个元素地释放
    ```
    
- `warning: deleting object of abstract class type ‘Shape’ which has non-virtual destructor will cause undefined behavior [-Wdelete-non-virtual-dtor]`
    
    这个报错是什么意思呢？去检查一下你的基类Shape，其中的析构函数是否是虚函数？那个就是问题所在。因为你通过基类指针删除派生类对象时，由于`~Shape()` 不是虚函数，编译器只会调用 `~Shape()` 而不会调用 `~Circle()` 。这就导致Circle中的资源没有被正确释放，造成内存泄漏。
    
    <aside>
    💡
    
    **注意：只要一个类会被用作基类使用，就应该给它一个virtual析构函数。**
    
    </aside>
