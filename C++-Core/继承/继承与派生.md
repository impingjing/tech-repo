# 继承与派生
---

## 继承与派生的相关概念

- **基类（父类）**
- **派生类（子类）**  派生类是通过对基类进行扩充和修改得到的。基类的所有成员自动成为派生类的成员。
- **继承（Inherence）**代表了从属（**is-a**）关系  比如：Student is a Person

## 为什么需要继承？

### 继承提高程序的可重用性

- 派生类重用基类类的代码可提高程序开发效率。
- 这是一种灵活的重用方式：子类在继承父类代码的基础上，可根据自己的特性进行调整。

## 多种继承关系

### 单继承

派生类只有一个直接基类

### 多重继承

派生类具有两个及以上直接基类

### 菱形继承

多重继承的特殊形式：派生类两次及以上重复继承某个祖先类（常涉及到 **虚继承** 的问题。）

## 成员访问控制与继承

### private（私有）

private后声明的成员称为私有成员，私有成员只能通过本类的成员函数来访问。**不可被继承！**

### public（公有）

在public后声明的成员称为公有成员，公有成员描述一个类与外部世界的接口，类的外部（程序的其它部分的代码）可以访问公有成员。**可以被继承！**

### protected（受保护）

受保护成员具有private与public的双重角色。类的外部不可以访问，如同private；派生类可以访问，如同 public。可以被继承，派生类的成员函数可以访问。即：**protected成员只能由本类及其直接/间接派生类的成员函数访问。**

## 继承访问控制

- 影响继承成员（派生类从基类继承而来的成员）访问控制方式的两个因素：
    - 定义派生类时指定的继承访问控制
    - 该成员在基类所具有的成员访问控制

| 基类中成员的访问控制 | 继承访问控制 | 派生类中继承成员的访问控制 |
| --- | --- | --- |
| public/protected/private | public | public/protected/不可访问 |
| public/protected/private | protected | protected/protected/不可访问 |
| public/protected/private | private | private/private/不可访问 |
- 私有和不可访问有区别

私有成员可以由派生类本身访问，不可访问成员即使是派生类本身也不能访问。

## 构造与析构

### 初始化与构造函数

- **基类的构造函数和析构函数不被继承**，派生类中需要声明自己的构造函数和析构函数
- **派生类**的构造函数**只需要**对**本类中新增成员进行初始化**即可
- 如果派生类构造函数未显式指定基类构造函数，编译器会在构造函数初始化列表中自动插入对基类默认构造函数（或无参拷贝构造函数）的调用
    - 如果基类没有默认构造（包括 =delete），则编译错误
- 派生类的构造函数需要使用基类的有参构造函数，必须显式在**初始化列表**中申明。注意：不能在构造函数内调用！

### 构造与析构调用顺序

- 构造：先基类再派生类
- 析构：先派生类再基类

### 程序示例

```cpp
class Person{
protected:
    string name;
    int age;
public:
    Person(string n,int a):name(n),age(a){ cout<<"创建了一个人类"<<endl; }
    ~Person(){cout<<"销毁一个人类"<<endl; }
}；
class Student:public Person{
private:
    double gpa;
public:
    Student(string n,int a,double g):Person(n,a),gpa(g)
    { cout<<"创建了一个学生"<<endl; }
    ~Student(){ cout<<"销毁了一个学生"<<endl; }
}
int main(){
    Student student1("Tom",19,4.1);
}

/*调用结果:
创建了一个人类
创建了一个学生
销毁了一个学生
销毁了一个人类
*/
```

### 经典问题

1. （若基类构造函数带参数，则定义派生类构造函数时，仅能通过初始化列表显式调用基类构造函数，并向基类构造函数传递实参。）为什么只能通过初始化列表？

派生类对象在进入构造函数函数体之前必须先把“基类那一部分”构造好，而基类构造函数的参数只能在构造开始时传进去，所以只能在初始化列表里显式调用带参的基类构造函数，不能在函数体里再“补调用”。

1. 基类构造函数调用是否可以忽略？

```cpp
// 以下三种情况可以忽略：
Base();
Base(int x=0);
Base()=default;
```

## 派生与成员函数

### 重载VS重写VS隐藏

| **维度** | **重载 (Overloading)** | **隐藏 (Hiding / Shadowing)** | **重写/覆盖 (Overriding)** |
| --- | --- | --- | --- |
| **发生的作用域** | **同一个**类中（或同级作用域） | **不同**的作用域（基类与派生类之间） | **不同**的作用域（基类与派生类之间） |
| **函数名字** | 必须相同 | 必须相同 | 必须相同 |
| **参数列表** | **必须不同** | 相同或不同皆可 | **必须完全相同** |
| **基类 virtual 关键字** | 无关 | 不需要（或签名不同时的 virtual） | **必须有** `virtual` 关键字 |
| **多态行为** | 静态绑定（编译期决定） | 静态绑定（编译期决定） | 动态绑定（运行期决定，多态的核心） |
- **隐藏**：
    
    (1) 派生类的函数与基类的函数同名，但是参数列表有所差异。
    (2) 派生类的函数与基类的函数同名，参数列表也相同，但是基类函数没有virtual关键字。
    

```cpp
//重写
class Animal{
public:
    virtual void cry(){ cout<<"……"<<endl; }
};
class Dog:public Animal{
public:
    void cry(){ cout<<"汪汪"<<endl; }
};
class Cat:public Animal{
public:
    void cry(){ cout<<"喵喵"<<endl; }
};
int main(){
    Dog dog;
    Animal* animal=&dog;
    animal->cry(); //猜猜看，会打印出来什么？
    // 打印结果：汪汪
    return 0;
}
//隐藏
class Animal{
public:
    void cry(){ cout<<"……"<<endl; }
};
class Dog:public Animal{
public:
    void cry(){ cout<<"汪汪"<<endl; }
    // 如果想要调用被隐藏函数，我们可以通过::作用域的方式进行
    // void cry(){ Animal::cry(); cout<<"汪汪"<<endl; }
};
int main(){
    Dog dog;
    Animal* animal=&dog;
    animal->cry(); //猜猜看，会打印出来什么？、
    // 打印结果：……
    return 0;
}

/*为什么没有打印出来“汪汪”？
虽然 animal 实际上指向的是 Dog 对象，但因为 cry() 不是虚函数，发生了静态绑定。
编译器只看 animal 的定义类型（Animal*），直接锁定了 Animal::cry()。
此时，派生类中的 cry() 函数被隐藏了，仿佛不存在一样。*/
```

### 对比：加上virtual会发生什么？

| 特性 | 非虚函数 (无 virtual) | 虚函数 (有 virtual) |
| --- | --- | --- |
| 绑定时间 | 编译时 (静态绑定) | 运行时 (动态绑定) |
| 决定因素 | 看指针的类型 (`Animal*`) | 看指针实际指向的对象 (`Dog`) |
| 行为 | 调用基类版本 (发生隐藏) | 调用派生类版本 (发生重写/多态) |

### 虚函数实现多态的原理

没有 `virtual` 时，C++ 采用的是**静态绑定**。编译器只看指针或引用的类型，在编译时就确定了调用哪个函数。而多态的目标是实现**动态绑定**，即程序在运行时，根据指针或引用**实际指向的对象类型**，来决定调用哪个函数版本。

#### **⚙️ 实现机制：vtable 与 vptr 的协作**

为了实现动态绑定，编译器在幕后为我们做了两件事：

1. **为每个含有虚函数的类生成一张“虚函数表” (vtable)**
    - 这张表可以看作是一个**函数指针数组**，存储在程序的只读数据区。
    - 表中存放了该类所有虚函数的地址。
    - 如果派生类重写了基类的虚函数，那么在派生类的 vtable 中，对应位置的函数地址就会被替换成派生类版本的地址。
2. **为每个含有虚函数的对象添加一个“虚表指针” (vptr)**
    - 这是一个隐藏在对象内存布局最前端的指针。
    - 当对象被创建时，它的构造函数会自动初始化这个 `vptr`，让它指向自己所属类的那个唯一的 `vtable`。

#### 🔎调用流程

假设代码如下：

```cpp
Base* ptr = new Derived(); // Base 中有 virtual void show()
ptr->show();
```

编译器生成的指令会执行以下三步“查表”操作，而不是直接跳转：

1. **找到 vptr**：首先，通过 `ptr` 找到它指向的对象在内存中的起始地址。因为 `vptr` 总是在对象的最前面，所以可以立即获取到这个 `vptr`。
2. **找到 vtable**：顺着 `vptr` 的指引，找到该对象所属类（这里是 `Derived`）的 `vtable`。
3. **找到并调用函数**：在 `vtable` 中，根据 `show()` 函数的固定索引（比如第0项），找到对应的函数地址。这个地址已经是 `Derived::show()` 的地址了，最后进行间接调用。

#### 💡为什么析构函数必须是虚函数？

**当一个类可能被当作基类使用时，它的析构函数必须声为 `virtual`。**

当你执行 `delete ptr;` 时，如果析构函数不是虚函数，就会发生静态绑定，只会调用 `Base` 类的析构函数。这意味着 `Derived` 对象中特有的资源（如动态分配的内存）将无法被释放，从而导致**内存泄漏**。

而如果析构函数是虚函数，`delete` 操作就会触发动态绑定，通过 `vptr` 找到并调用 `Derived` 的析构函数，确保整个对象被完整、正确地清理。

## using

### 让隐藏显现

```cpp
class Base{
public:
    void print(){……}
    void print(int val){……}
}；

// 场景：派生类定义了与基类同名但参数不同的函数（非虚函数）

class Son : public Base {
public:
    void print(double val) { ... }
};

// 核心规则：同名隐藏
// 1. 派生类一旦定义了同名函数 print，基类所有的 print 版本都会被“隐藏”（不可见）。
// 2. 调用 s.print() -> 编译报错（Son 中无此签名，且无法去 Base 查找）。
// 3. 调用 s.print(3) -> 调用 Son::print(double)。因为 int 3 隐式转换为 double。
//    (注意：这里基类的 print(int) 完全被忽略，不会发生重载)

// 解决方案：using 声明
class Son : public Base {
public:
    using Base::print; // 将基类的 print 名字引入当前作用域
    void print(double val) { ... }
};

// 结果：
// 现在 Son 中既有 Base 的 print，也有自己的 print。
// 编译器会根据参数类型进行正常的“重载解析”。
```

### 恢复访问控制方式

基类中的`public`或`protected`成员，因使用`protected`或`private`继承访问控制而导致在派生类中的访问方式发生改变，可以使用“**访问
声明**”恢复为原来的访问控制方式

- 访问声明的形式：`using 基类名::成员名；`（放于适当的成员访问控制后）

```cpp
class Base{
public:
    int i;
    ……
}；
class Son:private Base{
public:
    using Base::i; // i的权限变为public,后续在类外可以通过派生类对象进行访问
    ……
}；
```
