# 最长回文子串

## 题目再现

> 给你一个字符串 `s`，找到 `s` 中最长的 回文 子串。
> 

## 程序实现

## 暴力解法：超出时间限制

```cpp
class Solution {
public:
    bool is_pal(const string sub){
        int l=sub.length();
        int left=0,right=l-1;
        while(left<right){
            if(sub[left]==sub[right]){
                left++;
                right--;
            }
            else{
                return false;
            }
        }
        return true;
    }
    string longestPalindrome(string s) {
        int l=s.length(); //字符串的总长度
        if(l<=1){ return s;} //处理空字符串以及单个字符的情况

        string sub; //子串
        for(int i=l;i>1;i--){
            //从最长子串开始找
            for(int idex=0;idex<=l-i;idex++){
                sub=s.substr(idex,i);
                if(is_pal(sub)){
                    return sub;
                }
            }
        }
        return s.substr(0,1);
    }
};
```

### ⏱️ 时间复杂度是 O(n³)

我们可以把你的代码拆解为三层嵌套的“循环”：

1. **外层循环（子串长度）**：`for(int i=l; i>1; i--)`，遍历所有可能的子串长度，耗时 **O(n)**。
2. **中层循环（起始位置）**：`for(int idex=0; idex<=l-i; idex++)`，遍历该长度下的所有起始位置，耗时 **O(n)**。
3. **内层操作（回文判断）**：`is_pal(sub)` 函数内部有一个 `while` 循环，需要逐个字符比对。此外，`s.substr(idex, i)` 每次都会创建一个新的字符串副本，这也是一个 **O(n)** 的操作。

综合起来，总时间复杂度就是 **O(n) × O(n) × O(n) = O(n³)**。

## 中心扩展：时间复杂度为 $O(n^2)$

```cpp
class Solution {
public:
    string longestPalindrome(string s) {
        // 处理空字符串和单个字符的情况
        if(s.length()<2){ return s; }
        // 用中心扩展的方法优化运行时间
        int start=0; //记录最长回文子串的起始位置
        int end=0;   //记录最长回文子串的结束位置

        for(int i=0;i<s.length();i++){
            // 1.奇数长度回文
            int len1=expandAroundCenter(s,i,i);
            // 2.偶数长度回文
            int len2=expandAroundCenter(s,i,i+1);

            int maxLen=max(len1,len2); //取两种情况的最长长度
            if(maxLen>end-start){
                // 如果找到更大长度的回文子串，就更新起始、终止位置
                start=i-(maxLen-1)/2;
                end=i+maxLen/2;
            }
        }

        return s.substr(start,end-start+1);
    }
private:
    int expandAroundCenter(const string& s,int l,int r){
        while(l>0 && r<s.length() && s[l]==s[r]){
            l--;
            r++;
        }
        return r-l-1;
    }
};
```

### 为什么优化了运行时间？

中心扩展法利用了回文串**“中心对称”**的绝对规律。它不需要去猜子串的起始位置和长度，而是直接锁定“中心”：

1. **第1层循环**：遍历字符串中的每一个位置，把它当作中心（因为一个长度为 n 的字符串，只有 2n-1 个可能的中心点，属于 O(n) 级别）。
2. **第2层操作**：从这个中心向两边“扩散”（即你的 `expandAroundCenter` 函数）。在扩散的过程中，**一边移动指针，一边就在做回文判断了**。

它把暴力法中独立的“枚举子串”和“判断回文”这两步，完美地合并成了“从中心扩散”这一个动作。
